# LV2 Host Compatibility & Stability Fixes

## Summary

A catalogue stress test revealed several host-side bugs that caused crashes or mis-behaviour
when loading real-world LV2 plugins. All fixable issues have been resolved. Remaining
failures are due to broken plugin metadata or missing display support, which cannot be
fixed in the host.

**Commit:** `664eb64` — `lv2: fix host bugs found by catalogue stress test`

**Test coverage:** `tests/test_lv2plugin.cpp` — 19 GTest tests (18 lifecycle + 1 fork-isolated
catalogue stress test across all installed plugins).

**Before / after:**  816 plugins succeed, 57 crash (all in unfixable categories).

---

## Change 1 ✓ FIXED: Calf plugins crash on unload (SIGSEGV in dlclose)

**File:** `src/LV2Plugin.hpp` — `LV2Plugin::closePlugin()`

Calf plugins (Vinyl, Wavetable, etc.) spawn internal pthreads at instantiation time.
`lilv_instance_free()` calls `dlclose()`, which unmaps the DSO while those threads
are still executing code inside it — causing an immediate SIGSEGV.

**Fix:** Before calling `lilv_instance_free()`, re-open the plugin's shared library
with `RTLD_NODELETE` to permanently pin it in the process address space.  The DSO
will never be unloaded again, but the threads can keep running safely.

```cpp
// Pin the DSO permanently so in-flight plugin threads don't crash after dlclose
if (!lib_uri.empty()) {
    std::string lib_path = lilv_uri_to_path(lib_uri.c_str());
    dlopen(lib_path.c_str(), RTLD_LAZY | RTLD_NODELETE);
}
lilv_instance_free(instance_);
```

## Change 2 ✓ FIXED: Output atom port capacity not initialised (gareus/bitmeter abort)

**File:** `src/LV2Plugin.hpp` — `init_ports()` and `process()` Step E

The LV2 spec requires the host to set `atom.size` on **output** atom ports to the
available capacity (i.e. `buf_size - sizeof(LV2_Atom)`) before every `run()`, so the
plugin knows how many bytes it may write.  The code was setting `atom.size = 0` on
init and then `required_atom_size_` (the full buffer size, not the capacity) on reset —
both wrong.  gareus/bitmeter asserts `capacity > 920` and aborted immediately.

**Fix:** Use `p.atom_buf_size - sizeof(LV2_Atom)` for output atom port `atom.size` in
both the initial allocation and the per-cycle reset:

```cpp
// init_ports() — output atom port:
p.atom->atom.size = p.atom_buf_size - sizeof(LV2_Atom);   // was: = 0

// process() Step E — per-cycle reset:
p.atom->atom.size = p.atom_buf_size - sizeof(LV2_Atom);   // was: required_atom_size_
```

## Change 3 ✓ FIXED: Missing `lv2:uri-map` legacy feature (arpeggiator, sineshaper, etc.)

**File:** `src/LV2Plugin.hpp` — `init_features()` and `init_instance()`

Old plugins built with `lv2-c++-tools` (arpeggiator, sineshaper, control2midi, …) use
the deprecated `http://lv2plug.in/ns/ext/uri-map` feature.  They do not declare it as
`lv2:requiredFeature`, so `checkFeatures()` passes.  But at instantiation time their
constructor calls `uri_to_id()` through the feature data pointer — which is `nullptr`
because the host only provided `lv2:urid:map`.  This caused SIGABRT / SIGSEGV.

**Fix:** Implement `LV2_URI_Map_Feature` by delegating to our existing URID map, and
add it to the features array passed to `lilv_plugin_instantiate()`:

```cpp
// in Features struct:
LV2_URI_Map_Feature uri_map_data_;
LV2_Feature         uri_map_f;

// init_features():
uri_map_data_.callback_data = this;
uri_map_data_.uri_to_id = [](LV2_URI_Map_Callback_Data data,
                              const char* /*map*/, const char* uri) -> uint32_t {
    return static_cast<LV2Plugin*>(data)->map_uri(uri);
};
uri_map_f = { LV2_URI_MAP_URI, &uri_map_data_ };
```

All deprecation warnings suppressed with `#pragma GCC diagnostic` guards around
`<lv2/uri-map/uri-map.h>` and `<lv2/event/event.h>` includes and usage sites.

## Change 4 ✓ FIXED: `lv2:EventPort` ports left unconnected (control2midi, rudolf556)

**File:** `src/LV2Plugin.hpp` — `init_ports()`, `init_instance()`, `process()`, `closePlugin()`

Old-style `lv2:event#EventPort` ports (the precursor to `atom:AtomPort`) were not
detected and left completely unconnected.  After the uri-map fix allowed these plugins
to instantiate, they crashed in `run()` when dereferencing the null port pointer.

**Fix:** Detect event ports, allocate a minimal `LV2_Event_Buffer` (4096-byte data
area), connect the port to it, reset it each cycle, and free it on unload:

- `init_ports()`: detect with `lilv_port_is_a(..., event_class_)`, allocate:
  ```cpp
  p.event_buf_raw = (uint8_t*)calloc(sizeof(LV2_Event_Buffer) + 4096, 1);
  auto* evbuf = reinterpret_cast<LV2_Event_Buffer*>(p.event_buf_raw);
  evbuf->data       = p.event_buf_raw + sizeof(LV2_Event_Buffer);
  evbuf->header_size = sizeof(LV2_Event_Buffer);
  evbuf->stamp_type  = LV2_EVENT_AUDIO_STAMP;
  evbuf->capacity    = 4096;
  evbuf->size        = evbuf->event_count = 0;
  ```
- `init_instance()`: `lilv_instance_connect_port(instance_, p.index, p.event_buf_raw)`
- `process()` Step E: reset `event_count = 0; size = 0` for input ports each cycle
- `closePlugin()`: `free(p.event_buf_raw)`

## Change 5 ✓ FIXED: Unrecognised ports left dangling

**File:** `src/LV2Plugin.hpp` — `init_instance()`

Ports that are neither audio, control, atom, nor event were previously left
unconnected (and logged as a warning).  The LV2 spec requires every port to be
connected before `activate()`.  Changed to connect them to `nullptr`:

```cpp
if (!p.is_atom && !p.is_control && !p.is_event)
    lilv_instance_connect_port(instance_, p.index, nullptr);
```

---

## Change 6 ✓ DONE: Remove always-on sanitizers

**Files:** `CMakeLists.txt`, `packaging/rpm/opiqo.spec`, `docs/tests.md`, `README-gtk4.md`

`libasan` / `libubsan` were unconditionally linked in the debug build, causing
significant CPU overhead at runtime.  Removed the sanitizer block from CMake and the
RPM spec.  Sanitizers can still be re-enabled manually if needed.
